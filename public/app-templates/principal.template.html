Práctica 5: Aplicaciones web en el lado del cliente



Casos de uso con Angular:

Aplicación angular alojada en el directorio public


  1. Cualquier visitante podrá registrar en cualquier momento un artista nuevo (que representa un usuario de la aplicación).
    • Ruta API Rest: POST /artista
    • Ruta Angular: #/registro

  2. Cualquier visitante podrá acceder al perfil de un artisto, pudiendo ver todos sus datos escepto su contraseña.
    • Ruta API Rest: GET /artista/:id
    • Ruta Angular: #/artista/:idArtista

  3. Cualquier visitante podrá acceder al perfil de datos de una canción, pudiendo ver todos sus datos.
    • Ruta API Rest: GET /cancion/:id
    • Ruta Angular: #/cancion/:idCancion

  4. Cualquier visitante podrá obtener un listado paginado de todos los artistas almacenados en el sistema, en cada resultado aparecerá el numero de página que se está devolviendo, el número total de páginas y una coleccion de de artistas del tamaño indicado en la URL. (no se devuelve ni sus canciones ni comentarios, ya que no se mostraran en la plantilla de lista de artistas)
    • Ruta API Rest: GET /artista/pagina/:num_pagina/:limit
    • Ruta Angular: #/artistas

  5. Cualquier visitante podrá obtener un listado paginado de todas las canciones almacenadas en el sistema, en cada resultado aparecerá el numero de página que se está devolviendo, el número total de páginas y una coleccion de de artistas del tamaño indicado en la URL. (no se devuelve ni sus canciones ni comentarios, ya que no se mostraran en la plantilla de lista de artistas)
    • Ruta API Rest: GET /cancion/pagina/:num_pagina/:limit
    • Ruta Angular: #/canciones

  6. Cualquier visitante podrá obtener un listado paginado de todas las canciones de un artista almacenadas en el sistema, en cada resultado aparecerá el numero de página que se está devolviendo, el número total de páginas y una coleccion de de artistas del tamaño indicado en la URL. (no se devuelven sus comentarios, ya que no se mostraran en la plantilla de lista de canciones)
    • Ruta API Rest: GET /artista/:id/canciones/pagina/:num_pagina/:limit
    • Ruta Angular: #/artista/:idArtista/canciones

  7. Cualquier visitante podrá obtener un listado paginado de todos los comentarios de un artista o de una canción almacenados en el sistema, en cada resultado aparecerá el numero de página que se está devolviendo, el número total de páginas y una coleccion de de artistas del tamaño indicado en la URL.
    • Ruta API Rest Artista: GET /artista/:id/comentarios/pagina/:num_pagina/:limit
    • Ruta API Rest Cancion: GET /cancion/:id/comentarios/pagina/:num_pagina/:limit
    • Ruta Angular Artista: #/artista/:idArtista (Los comentarios se ven en el perfil)
    • Ruta Angular Artista: #/cancion/:idCancion (Los comentarios se ven en el perfil)

  8. Cualquier visitante podrá añadir un nuevo comentario a un artista o a una canción.
    • Ruta API Rest Artista: POST /artista/:id/comentar
    • Ruta API Rest Cancion: POST /cancion/:id/comentar
    • Ruta Angular Artista: #/artista/:idArtista (Los comentarios añaden desde el perfil)
    • Ruta Angular Artista: #/cancion/:idCancion (Los comentarios añaden desde el perfil)

  9. Cualquier visitante podrá loguearse en cualquier momento como un artista siempre que conozca su nombre de usuario y contraseña.
    • Ruta API Rest: POST /autentificacion
    • Ruta Angular: El formulario de login se encuentra en la barra superior del menú.
    • Datos de logueo para probar la aplicación
      • User: TheDoors
      • Pass: TheDoors

  10. Un artista logueado podrá crear una canción, la cancion creada se guardará como propiedad del artista logueado, no se pueden crear canciones en otros artistas y la propiedad de la cancion no se puede modificar. (Se debe mandar en token de autorizacion en la cabecera)
    • Ruta API Rest: POST /cancion
    • Ruta Angular: #/publicar-cancion

  11. Un artista logueado podrá modificar una canción siempre y cuando sea de su propiedad. (Se debe mandar en token de autorizacion en la cabecera)
    • Ruta API Rest: PUT /cancion/:id
    • Ruta Angular: #/modificar-cancion/:idCancion

  12. Un artista logueado podrá eliminar una canción siempre y cuando sea de su propiedad. (Se debe mandar en token de autorizacion en la cabecera)
    • Ruta API Rest: DELETE /cancion/:id
    • Ruta Angular: #/eliminar-cancion/:idCancion

  13. Un artista logueado podrá modificar sus propios datos, pero no los de otro artista. (Se debe mandar en token de autorizacion en la cabecera)
    • Ruta API Rest: PUT /artista/:id
    • Ruta Angular: #/modificar-perfil

  14. Un artista logueado podrá darse de baja a si mismo, pero no eliminar otros artistas. (Se debe mandar en token de autorizacion en la cabecera)
    • Ruta API Rest: DELETE /artista/:id
    • Ruta Angular: #/baja
